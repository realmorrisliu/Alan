# Plans Index

Plans are execution sequencing documents, not source-of-truth contracts.

Use this directory for:

1. Active implementation plans.
2. Migration sequencing notes that still guide near-term work.

If a plan stops guiding current work and its substance is already captured by
newer specs or guides, delete it instead of keeping a historical archive in the
repository. Git history is sufficient for old rollout context.
