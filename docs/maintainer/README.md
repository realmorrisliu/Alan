# Maintainer Docs

This directory is for maintainer-facing repository operations and project notes.

Use it for:

1. Evergreen repository setup and automation instructions.
2. Project-specific maintainer notes that do not belong in the public spec
   directory.
3. Rollout notes that are still actively useful to maintainers.

Do not use it for:

1. Source-of-truth runtime contracts.
2. Speculative ideas.
3. Historical rollout trackers that no longer guide current work.
